# 从零到上线：用 React + Supabase 做一个待办 App

**作者：菠萝芯**
**日期：2026年3月14日**
**网址：khao.tech**

---

## 概述

本教程记录了如何从零开始，用现代前端技术栈构建并部署一个具备用户登录、云端数据同步功能的待办清单 App，并绑定自定义域名上线供全国用户使用。

---

## 技术栈

| 技术 | 用途 |
|------|------|
| React | 前端 UI 框架 |
| TypeScript | 类型安全的 JavaScript |
| Vite | 前端构建工具 |
| Supabase | 后端数据库 + 用户认证 |
| GitHub | 代码托管 |
| Vercel | 自动部署 + CDN |
| 阿里云 | 域名购买与 DNS 解析 |

## 第一步：创建项目

使用 Vite 脚手架快速创建 React + TypeScript 项目：

```bash
npm create vite@latest todo-app -- --template react-ts
cd todo-app
npm install
```

启动本地开发服务器：

```bash
npm run dev
```

浏览器访问 `http://localhost:5173` 即可预览。

## 第二步：开发待办功能

### 核心数据结构

```typescript
interface Todo {
  id: string
  text: string
  done: boolean
  priority: 'low' | 'medium' | 'high'
  category: string
  due_date: string | null
}
```

### 功能列表

- 添加任务（按回车或点添加按钮）
- 点击任务标记完成/未完成
- 删除任务
- 设置优先级（高/中/低，用颜色区分）
- 设置分类标签
- 设置截止日期（逾期自动标红）
- 筛选全部/未完成/已完成
- 统计完成情况

### localStorage 本地持久化（早期版本）

```typescript
const [todos, setTodos] = useState<Todo[]>(() => {
  const saved = localStorage.getItem('todos')
  return saved ? JSON.parse(saved) : []
})

useEffect(() => {
  localStorage.setItem('todos', JSON.stringify(todos))
}, [todos])
```

## 第三步：接入 Supabase（用户登录 + 云端存储）

### 3.1 创建 Supabase 项目

1. 访问 [supabase.com](https://supabase.com)，用 GitHub 账号登录
2. 点击 **New Project**
3. 填写项目名称、数据库密码，选择区域（新加坡）
4. 等待约1分钟创建完成

### 3.2 创建数据库表

在 Supabase **SQL Editor** 运行以下 SQL：

```sql
create table todos (
  id uuid default gen_random_uuid() primary key,
  user_id uuid references auth.users not null,
  text text not null,
  done boolean default false,
  priority text default 'medium',
  category text default '',
  due_date date,
  created_at timestamp with time zone default now()
);

-- 开启行级安全，每个用户只能看到自己的数据
alter table todos enable row level security;

create policy "Users can access their own todos" on todos
  for all using (auth.uid() = user_id);
```

### 3.3 安装 Supabase 客户端

```bash
npm install @supabase/supabase-js
```

### 3.4 配置环境变量

创建 `.env.local` 文件（不会被提交到 GitHub）：

```
VITE_SUPABASE_URL=你的项目URL
VITE_SUPABASE_ANON_KEY=你的anon公钥
```

### 3.5 初始化 Supabase 客户端

```typescript
// src/lib/supabase.ts
import { createClient } from '@supabase/supabase-js'

export const supabase = createClient(
  import.meta.env.VITE_SUPABASE_URL,
  import.meta.env.VITE_SUPABASE_ANON_KEY
)
```

### 3.6 用户认证

```typescript
// 注册
await supabase.auth.signUp({ email, password })

// 登录
await supabase.auth.signInWithPassword({ email, password })

// 退出
await supabase.auth.signOut()

// 监听登录状态
supabase.auth.onAuthStateChange((_event, session) => {
  setUser(session?.user ?? null)
})
```

### 3.7 配置邮件跳转地址

Supabase → **Authentication → URL Configuration**：
- **Site URL**：`https://khao.tech`
- **Redirect URLs**：`https://khao.tech`

## 第五步：Vercel 部署

1. 登录 [vercel.com](https://vercel.com)
2. 点击 **Add New Project**，选择 GitHub 仓库
3. 进入 **Settings → Environment Variables**，添加：
   - `VITE_SUPABASE_URL`
   - `VITE_SUPABASE_ANON_KEY`
4. 点击 **Redeploy**

每次推送代码到 GitHub，Vercel 会自动重新部署。

---

## 第六步：绑定自定义域名

### 在 Vercel 添加域名

项目 → **Settings → Domains** → 输入 `khao.tech` → Add

Vercel 会提供一条 DNS 记录，例如：
```
类型：A
名称：@
值：76.76.21.21
```

### 在阿里云配置 DNS

1. 登录阿里云 → 域名控制台 → 云解析 DNS
2. 点击 **添加记录**
3. 填写：
   - 记录类型：`A`
   - 主机记录：`@`
   - 记录值：`76.76.21.21`
4. 点击确定

DNS 生效后（约5-10分钟），访问 `https://khao.tech` 即可看到上线的 App。

## 总结

从一个空文件夹，到一个全国可访问、支持用户登录、数据云端同步的 App，核心流程是：

**写代码 → 推送 GitHub → Vercel 自动部署 → 域名绑定**

整个过程不需要购买服务器，不需要运维，完全免费（Supabase 和 Vercel 均有免费套餐），非常适合个人项目和初学者入门。
